## 2. 設計原則

### 2.1 ターゲットユーザー特性

| ユーザー | 特性 | 配慮事項 |
|----------|------|----------|
| スタッフ | 業務中・忙しい | 最小タップで入力完了 |
| 家族 | 高齢者含む | 大きなタッチターゲット |
| 管理者 | 設定変更が多い | 一覧性・効率性 |

### 2.2 基本原則

1. **シンプルさ**: 1画面1目的
2. **視認性**: 大きなフォント・高コントラスト
3. **到達性**: 重要な操作はフッターから2タップ以内
4. **フィードバック**: 操作結果を即時表示

## 4. モーダル・フォーム設計

### 4.1 設定モーダル

**詳細仕様**: [SETTINGS_MODAL_UI_SPEC.md](./SETTINGS_MODAL_UI_SPEC.md)

- ボタン配置: キャンセル（左） / 保存（右）
- クリアリンク: 左下に配置
- 確認ダイアログ: 破壊的操作時に表示

### 4.2 フォーム入力

| 要素 | 仕様 |
|------|------|
| 入力フィールド | 44px以上の高さ |
| ラベル | フィールド上部に配置 |
| エラー | フィールド下部に赤字表示 |
| 必須マーク | ラベル右に `*` |

## 5. テーブルビュー設計

**詳細仕様**: [TABLE_VIEW_COLUMNS.md](./TABLE_VIEW_COLUMNS.md)

### 5.1 基本原則

- モバイルファースト（スクロール可）
- 固定列: 日時・名前
- 優先表示: 重要度の高い情報を左に

### 5.2 シート別カラム数目安

| シート | 表示カラム数 | 備考 |
|--------|-------------|------|
| 食事 | 5-6 | 主食・副食・汁物 |
| バイタル | 4-5 | 体温・血圧・脈拍 |
| 排泄 | 3-4 | 時間・種別・状態 |

## 6. 家族向けUX設計

**詳細仕様**: [FAMILY_UX_DESIGN.md](./FAMILY_UX_DESIGN.md)

### 6.1 3つのビュー

```
┌─────────────────────────────────────────────────┐
│          遠隔ケア・コックピット                   │
├─────────────────────────────────────────────────┤
│                                                 │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐        │
│  │ View A  │  │ View B  │  │ View C  │        │
│  │エビデンス│  │ケア仕様 │  │ダッシュ │        │
│  │モニター │  │ビルダー │  │ボード   │        │
│  └─────────┘  └─────────┘  └─────────┘        │
│                                                 │
└─────────────────────────────────────────────────┘
```

| ビュー | 目的 |
|--------|------|
| View A | 写真エビデンスでケア状況を確認 |
| View B | 一時的なケア指示の変更・追記 |
| View C | 品物・統計のダッシュボード |

## 7. レスポンシブ対応

| ブレークポイント | 幅 | 対応 |
|------------------|-----|------|
| モバイル | < 640px | 1カラム・縦積み |
| タブレット | 640-1024px | 2カラム |
| デスクトップ | > 1024px | 3カラム以上 |

---

## 8. アクセシビリティ

| 項目 | 対応 |
|------|------|
| コントラスト比 | WCAG AA準拠（4.5:1以上） |
| タッチターゲット | 44x44px以上 |
| フォーカス表示 | リング表示 |
| スクリーンリーダー | aria-label付与 |
